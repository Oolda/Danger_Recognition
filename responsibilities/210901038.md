## 2. Initial XML Label Generation and Evaluation Support

I also worked on the XML label generation part at the beginning of the evaluation process.

My first version parsed the XML annotation files, created frame-level labels, and converted them into sequence-level labels using the same 10-frame sliding-window idea used by the temporal models.

This first version was not used directly as the final label file because we later identified two issues:

* The camera order in my XML sorting did not exactly match the order used in the spatial feature extraction pipeline.
* The first labeling logic was more general, while the final version needed to check only specific weapon classes such as handgun, short rifle, and knife.

Since the reconstruction errors and labels must follow the exact same frame order, this ordering issue could affect the reliability of the evaluation.

Although my first version was not the final one, it helped us identify the alignment problem and clarify how the final XML label generation should be implemented.

---

## 3. Verification of Label and Sequence Alignment

After noticing the ordering issue, I helped check the alignment between XML annotations, temporal sequences, reconstruction-error outputs, and evaluation labels.

This was important because the model outputs one reconstruction score per sequence, so the label array must match the exact same sequence order.

This work included checking:

* XML file ordering
* Camera-boundary preservation
* 10-frame sequence construction
* Sequence-label shape
* Consistency between labels and reconstruction outputs

This verification step helped make the final evaluation process more reliable and consistent.
